# Moongate Calibration Audit

Scope: Northstar onboarding and task-generation quality for `https://www.moongate.one/`.

## What Northstar generated

### Company understanding
- Company summary: Moongate appears to be a Solana wallet and trading product focused on making onchain activity faster and simpler.
- ICP: Solana traders, onchain power users, and developers who want faster wallet onboarding and execution.
- Key pages discovered:
  - `https://www.moongate.one/`
  - `https://www.moongate.one/dapps`
- Opportunities:
  - The value proposition is fast and bold, but it still under-explains trust, wallet onboarding, and why this is better than existing Solana trading flows.
  - The site speaks fluently to crypto-native users, but it leaves room to explain product mechanics and credibility for less degen, high-value users or integration partners.
  - There is no obvious educational or ecosystem content surface yet, which limits SEO and narrative control in a crowded Solana market.

### Generated backlog
- Clarify why Moongate is the fastest path into Solana trading
- Build a search and content map for Solana wallet and trading intent
- Create an ecosystem credibility scan for Moongate
- Draft a blog brief on Solana trading onboarding friction
- Design an X post set for crypto-native launch momentum

### Artifact
- Blog brief: `Draft a blog brief on Solana trading onboarding friction`
- Direction: explain friction in Solana trading onboarding, then tie the narrative back to Moongate's trust and speed angle.

## What Northstar gets right

- It correctly classifies Moongate as a Solana-native product, not a generic SaaS site.
- The ICP is directionally right for the current website language.
- The opportunity list is tied to actual crypto friction points: trust, onboarding, credibility, and content gaps.
- The top task is concrete and useful for the business.
- The tone feels crypto-native instead of corporate or generic.

## Where it sounds generic or wrong

- The SEO and content tasks are better than the earlier version, but they still lean on generic growth-system language in places.
- The X post task is useful, but the framing is still broad enough to feel like a standard launch-marketing template if it is not tightened further.
- "Ecosystem credibility scan" is directionally useful, but it is still a bit abstract unless it names the specific Moongate comparison set.

## Where it confuses the business model

- The output mixes three adjacent audiences:
  - Solana traders
  - onchain power users
  - developers/integration partners
- That blend is reasonable for a first pass, but it can blur whether Moongate is mainly a trader acquisition product, a wallet UX product, or a developer integration surface.
- The current summary does not yet make that split explicit enough, so the board risks optimizing for a composite audience instead of one dominant motion.

## Where task generation is too vague

- `Design an X post set for crypto-native launch momentum` is the weakest task for specificity because it does not anchor to a clear distribution moment, product claim, or audience segment.
- `Create an ecosystem credibility scan for Moongate` needs sharper competitor framing to avoid becoming a generic research task.
- `Build a search and content map for Solana wallet and trading intent` is relevant, but still broad unless it is narrowed to the highest-intent queries Moongate actually wants.

## What should be fixed before live demos

1. Make the company summary more explicit about Moongate's primary growth motion.
2. Split trader acquisition from developer/integration messaging so the ICP is less blended.
3. Tighten the social task so it is tied to a concrete launch or weekly content moment.
4. Make the competitor task name the category peers it should compare against.
5. Keep the current crypto-native tone, but reduce any copy that feels like a generic growth playbook.

## Smallest fixes needed

1. Refine the company summary to say whether the product is primarily for traders, wallet users, or integrators.
2. Keep the homepage task at the top because it is the clearest conversion lever.
3. Reword the X post task so it reflects one specific audience and one specific product claim.
4. Replace broad SEO language with a narrower intent map for Solana wallet onboarding and execution.
5. Preserve the current tone, but make each task name and rationale more concrete.

## Priority order

1. Homepage copy suggestion
2. Search/content map
3. Ecosystem credibility scan
4. Blog brief on onboarding friction
5. X post set

## What is still mocked or weak

- Site understanding still depends on a lightweight heuristic crawl, so Moongate is being inferred from visible homepage and dapp-language rather than deeper product usage data.
- The content strategy is still rules-driven, not model-planned.
- The board does not yet know the real split between trader growth and integration-led growth.
- Social tasks are still the most templated part of the backlog.

## What should be tested next

1. Re-onboard Moongate and verify the summary still reads crypto-native without drifting into generic SaaS language.
2. Check whether the ICP should be split or narrowed before live demos.
3. Review the X post task and competitor scan for specificity.
4. Approve or reject the blog brief and confirm the approval loop still feels founder-controlled.
